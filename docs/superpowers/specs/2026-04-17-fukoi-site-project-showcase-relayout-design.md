# Fukoi Site Project Showcase Relayout Design

## Goal

在不改动首页整体品牌气质、不削弱当前暗色高级感的前提下，重做首页项目展示区的版式关系，解决当前“一大一小”“视觉重心失衡”“画面比例粗糙”的问题，让该区块更像一个经过统一编排的游戏工作室作品展示，而不是两个独立卡片的简单拼接。

## Problem Statement

当前首页项目区的问题不是单一尺寸问题，而是整组构图秩序不足：

- 当前两张项目图被实现为上下堆叠的两个独立卡片，而不是一组统一设计的作品区。
- 第一张战术像素项目图承担了过强的“主视觉”角色，第二张 `GOBLIN` 更像补图，形成明显的一大一小。
- 第一张图当前取景过满，坦克前景过于贴脸，画面压迫感强，削弱了环境、天空、烟雾和战场关系。
- 两张图的画框、边距、图文节奏和标题位置没有形成足够稳定的网格，因此整体观感偏粗糙。

## Design Direction

最终采用：

- **结构方向：`C2` Alternating Editorial**
- **第一张图取景方向：`B` Balanced**

这意味着首页项目区不再维持“两个独立卡片上下堆叠”的表达，而改成一整个被统一编排的作品展示模块。

## Chosen Layout

### Alternating Editorial Composition

项目区改为两条交错排列的项目行：

- 第一行：左侧为战术像素项目视觉，右侧为该项目文案块
- 第二行：左侧为 `GOBLIN` 文案块，右侧为 `GOBLIN` 视觉

这种交错关系的目的：

- 消除第一张图天然压住第二张图的“大主图 + 小补图”关系
- 让两条项目都保持等重，但又不做死板对称
- 让项目区更像欧美游戏 studio 或 portfolio 页面中的编辑式编排
- 将两条项目从“两个 card”提升为“一个 section 内的两组作品单元”

### Visual Rhythm

整组项目区需要形成统一的版式语法：

- 两条项目使用同一套画框圆角、边框强度、内边距和标题节奏
- 两条项目的视觉块高度需要接近，避免出现明显的主次塌陷
- 文案块与图片块之间采用清晰但克制的留白关系，而不是强对比切割
- 每条项目的标题、状态标签、描述和行动链接保持统一层级

## Tactical Image Crop

### Chosen Crop: Balanced

第一张战术像素项目图采用 `B` 版平衡取景，而不是当前过度贴近的特写。

目标：

- 保留坦克作为项目识别点
- 但不让坦克占据画面绝对主导
- 拉出更多天空、烟雾、地平线和火光关系
- 让它更像“项目画面”而不是“角色怼脸式海报”

具体要求：

- 缩小当前视觉中的坦克占比
- 降低右下角前景压迫感
- 维持画面张力，但把世界关系重新带回画框

## Goblin Treatment

`GOBLIN` 项目图不作为副图处理，而作为等权的第二作品单元：

- 保留其明亮、轻奇幻、角色驱动的视觉特征
- 放入与战术像素项目相同的画框系统
- 元信息、编号挂签和文案位置统一到同一节奏中
- 不再让它看起来像第一张图旁边的补充素材

## Responsive Behavior

### Desktop

桌面端采用交错编辑式结构：

- 第一行图左文右
- 第二行文左图右

整体应有明确的横向编排感，而不是简单卡片堆叠。

### Mobile / Tablet

移动端与平板端不能保留桌面交错结构的硬拷贝，而应折叠为统一的纵向作品流：

- 每个项目作为一个完整单元纵向排列
- 图片在上，文字在下
- 两个项目保持同一体量与同一节奏
- 不允许移动端再次出现第一项明显更大、第二项像补图的问题

## Content Treatment

本次以结构与视觉秩序为主，不重写项目主文案，但允许对项目区标题层级做轻微调整，使其更符合作品集语境。

可保留或轻调的元素：

- `PROJECT PORTFOLIO`
- `FEATURED TITLES`
- 项目状态标签
- 项目底部挂签 / 编号信息

但这些元素必须服务于统一的 section 编排，而不是继续强调第一张图的“海报主视觉”地位。

## Implementation Scope

本次实现仅聚焦首页项目展示区：

- 重做项目区 HTML 结构
- 调整项目区 CSS 布局与比例系统
- 调整第一张图的 `object-position` / `scale` 或等效视觉取景方式
- 统一两条项目的元信息、标题、文案和 CTA 节奏
- 适配桌面、平板和手机

## Files Expected

- `index.html`
- `index.css`
- `content.js`

如果当前 JS 中存在与 reveal 或 section 节奏强绑定的逻辑，需要同步检查：

- `index.js`

## Non-Goals

- 不重做首页 Hero
- 不修改公司信息模块
- 不修改页脚
- 不改动 `STUDIO` 页
- 不删除现有项目内容
- 不把项目区改成传统企业介绍卡片
- 不引入新的业务叙事或与用户确认范围外的新文案方向

## Acceptance Criteria

实现完成后，应满足以下标准：

- 访客不会再直接感知到“一大一小”的项目主次失衡
- 第一张图不再给人“坦克被放得太满”的压迫感
- 两个项目看起来属于同一套设计系统，而不是两个独立 card
- 项目区整体更像高品质游戏 studio 的 portfolio 展示
- 手机端与平板端仍然保持清晰、完整、不过长的阅读节奏
